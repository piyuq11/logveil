# Contributing

Thanks for helping improve `logveil`.

1. Open an issue for behavior changes or new categories of detection.
2. Use synthetic credentials in tests and examples.
3. Add regression tests for every detection rule.
4. Keep runtime dependencies at zero unless there is a strong reason to add one.
5. Run `python -m unittest discover -s tests` before opening a pull request.

Reports of false positives are especially valuable. Include the smallest safe example that reproduces the problem.
